# Roadmap

## Near term

- Improve compatibility heuristics for heavy SPA sites
- Detect external verification / anti-bot signals more precisely
- Add more real-world regression URLs to smoke and manual test notes
- Make `check --json` output easier for other tools to consume

## Mid term

- Add optional machine-readable recommendation summaries for hybrid tools
- Improve form interaction coverage beyond simple selector fill and click flows
- Reduce false positives on simple static sites with unusual layout metrics

## Long term

- Share compatibility fixtures with `pi-browser-hybrid-skill`
- Publish a stable public compatibility scoring contract
- Explore optional screenshot support once Obscura exposes a reliable implementation
